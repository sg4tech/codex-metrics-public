# Retrospective: фиксирование обходных путей без устранения причины

## Situation

При работе в воркдире обнаружилось, что `pytest` тестирует код основного репо, а не воркдира — потому что `.venv` симлинкован из основного репо и editable install указывает туда же.

## What Happened

Был найден обходной путь: `PYTHONPATH=src .venv/bin/python -m pytest ...`. Он работал — зафиксировали в memory как постоянное правило.

Только позже, когда пользователь спросил "зачем вообще так?", выяснилось что правильное решение — одна строка в `pyproject.toml`:

```toml
[tool.pytest.ini_options]
pythonpath = ["src"]
```

После этого `PYTHONPATH=src` prefix не нужен нигде.

## Root Cause

Первый найденный рабочий вариант был принят как решение, не был задан вопрос: "это обходной путь или это правильное место для этого решения?"

## 5 Whys

1. Почему `PYTHONPATH=src` попал в memory как постоянное правило?
   Потому что он решил симптом и тесты стали зелёными.

2. Почему не было поиска корневого fix?
   Потому что режим "сделать работающим" дал результат быстрее чем режим "сделать правильно".

3. Почему режим "сделать работающим" доминирует?
   Потому что нет явного триггера на вопрос "а почему это вообще нужно?"

4. Почему такого триггера нет?
   Потому что workaround и fix выглядят одинаково в моменте — оба устраняют проблему.

5. Почему это проблема?
   Потому что workaround создаёт когнитивный налог, накапливается, и становится частью "правил" — хотя правила не должны быть костылями.

## Retrospective

Когда что-то записывается в memory как "нужно всегда делать X" — это сигнал остановиться и спросить:

> "X — это правильное решение, или обходной путь вокруг чего-то что можно починить?"

Если X звучит как "всегда добавляй prefix / флаг / env-переменную" — почти наверняка это workaround, и нужно искать где конфигурировать это один раз.

## Permanent Changes

- Добавлен `[tool.pytest.ini_options] pythonpath = ["src"]` в `pyproject.toml`
- Запись в memory обновлена: описывает fix, а не workaround
- Добавлено правило в memory: перед фиксацией workaround — проверить, можно ли убрать причину

## Classification Of Follow-up

- Код/конфиг: `pyproject.toml` изменён, `PYTHONPATH=src` больше не нужен для pytest
- Правило для агента: зафиксировано в memory
- Ретро: этот файл
