# Renaming to `ai-agents-metrics` may improve public discoverability and long-term positioning

Single-hypothesis file.

### H-032 — Renaming to `ai-agents-metrics` may improve public discoverability and long-term positioning

- Status: `active`
- Priority: `medium`
- Created: `2026-04-09`
- Statement:
  - The current name `codex-metrics` ties the product to OpenAI Codex, which is one specific agent runtime and a diminishing brand.
  - Renaming to `ai-agents-metrics` may better reflect the actual scope (any AI coding agent), improve discoverability in AI-search and keyword searches, and remove a false dependency signal for users who do not use Codex.
  - If that is true, the rename would increase the addressable audience without changing the product itself.
- Why it matters:
  - GEN SEO and organic discoverability depend on the name appearing in searches like "ai agent cost tracking", "ai agent metrics", "track AI agent tokens". `ai-agents-metrics` matches these patterns; `codex-metrics` does not.
  - Claude Code, Gemini CLI, and other agents are growing faster than Codex. Locking the brand to one agent runtime creates a ceiling on adoption.
  - PyPI package names and CLI command names are hard to change later once users depend on them. The cost of renaming now (before significant PyPI distribution) is lower than the cost of renaming after.
- Expected upside:
  - better organic discoverability for users searching for agent-agnostic metrics tools
  - cleaner positioning: the tool works with any AI agent, not just Codex
  - consistent external name across repo, PyPI, and CLI command
  - removes confusion for users who see "codex" and assume OpenAI Codex is required
- Expected resource cost:
  - medium: requires coordinated changes across repo, PyPI, CLI, docs, README, llms.txt, and internal module if renamed
  - if internal Python module `codex_metrics` is kept as-is, cost is lower (external name change only)
  - if internal module is also renamed to `ai_agents_metrics`, cost is higher (full refactor of all imports and test files)
- Main risks or where this may be wrong:
  - the rename may break any existing integrations, scripts, or workflows that reference `codex-metrics` or `codex_metrics`
  - `ai-agents-metrics` is a generic name and may be harder to own in search results than a more distinctive brand
  - if PyPI package `codex-metrics` was already claimed or indexed, renaming requires users to update their dependencies
  - the private repo and internal tooling still uses `codex-metrics`; renaming only the public surface creates a split that must be maintained
- Alternatives considered:
  - keep `codex-metrics` and rely on README language and GitHub topics to signal agent-agnosticism
  - rename to something more distinctive and brand-able (e.g., `agentledger`, `tokentrail`) rather than a descriptive compound name
  - rename only the GitHub repo and README; keep PyPI package and CLI as `codex-metrics` for now
- Current confidence:
  - `medium`
- Current priority note:
  - medium priority; timing is good (before significant PyPI distribution), but the decision depends on whether the team wants a descriptive name or a distinctive brand
- Evidence status:
  - `codex-metrics` is not yet on PyPI, so the rename cost is near-minimum right now
  - GitHub repo is public but has low traffic; redirect will cover existing links
  - no confirmed external users who would be broken by the rename
- Open questions before committing:
  - keep internal Python module as `codex_metrics` or rename to `ai_agents_metrics`?
  - keep old `codex-metrics` CLI alias for a transition period or drop immediately?
  - rename the private repo and internal tooling at the same time, or only the public surface?
    **DECISION (permanent): the private repo will NEVER be renamed. It stays as `codex-metrics` permanently. This question is closed.**
- Evaluation plan:
  - decide on internal module rename scope
  - execute the rename in a single coordinated task
  - verify CI passes on the public repo after rename
  - verify `pip install ai-agents-metrics` and `ai-agents-metrics show` work end-to-end
  - update llms.txt, README, and docs to reflect the new name
  - publish to PyPI under the new name
