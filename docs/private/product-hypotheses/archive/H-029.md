# A symlink from CLAUDE.md to AGENTS.md may improve rule uptake by putting instructions directly in context

Single-hypothesis file.

### H-029 — A symlink from CLAUDE.md to AGENTS.md may improve rule uptake by putting instructions directly in context

- Status: `validated`
- Created: `2026-04-05`
- Statement:
  - Currently `CLAUDE.md` contains only a redirect ("read AGENTS.md") while the actual rules live in `AGENTS.md`. Claude Code loads `CLAUDE.md` into the system context automatically but reads `AGENTS.md` only if the agent follows the redirect instruction.
  - A symlink (`ln -s AGENTS.md CLAUDE.md`) makes `CLAUDE.md` transparently resolve to `AGENTS.md` content, so the full rule set is injected into system context automatically without a separate read step.
- Why it matters:
  - Every missed "read AGENTS.md" step is a rule-violation opportunity. If rules are already in context, the failure mode disappears structurally.
  - Structural guarantees are stronger than instructional ones: "file is in context" is harder to skip than "instruction says to read the file".
- Expected upside:
  - eliminate the class of failures where an agent starts a task without reading AGENTS.md
  - reduce context-setup overhead per session
- Expected resource cost:
  - low — one `ln -s` command, committed to the repo
- Main risks or where this may be wrong:
  - **Context overhead.** `AGENTS.md` is ~10 KB. Loading it on every session via `CLAUDE.md` increases per-session context cost vs. the current 186-byte redirect.
- Alternatives considered:
  - Keep the current redirect but enforce reading via a pre-task hook or validation step.
  - Merge `AGENTS.md` content directly into `CLAUDE.md` and delete `AGENTS.md` — single source of truth, no link mechanics. Downside: loses the `AGENTS.md` name convention used by other agents (e.g., OpenAI Codex, Cursor).
  - Use a hard link (`ln`): git tracks both files separately, worktrees don't inherit it — worse than symlink.
  - Add a Definition of Done check that verifies `AGENTS.md` was read at session start.
- Current confidence:
  - `high`
- Evidence status:
  - `2026-04-05`: experiment confirmed that Claude Code reads symlinks via filesystem I/O, which follows the symlink and returns full file content. Git stores the symlink as mode `120000` (symlink object), so it is committed and restored correctly in all worktrees on checkout — worktree concern is not valid.
- Next re-evaluation trigger:
  - after observing whether context overhead (~10 KB per session) causes measurable cost regression
- Related hypotheses:
  - relates to H-011 (active-task enforcement) — same class of problem: rules exist but are not structurally guaranteed to be in scope
- Notes:
  - `2026-04-05`: raised in conversation while reviewing CLAUDE.md/AGENTS.md redirect setup.
  - `2026-04-05`: experiment run, risks resolved, symlink implemented.
