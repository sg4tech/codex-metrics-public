# PM Best Practices — External Canon

Reference library of external product management best practices. This is **not** project-specific to `codex-metrics` — it is the broader industry canon, organized for scanning and selective deep-dives.

## How to use

- **Not mandatory reading.** This folder is optional reference for product framing, metrics interpretation, and discovery work. Consult the relevant file when a specific decision calls for it.
- **Names, not summaries.** Each file lists the canonical authors/works and the core frameworks they introduced. Read the source if the framework becomes load-bearing for a decision — this library does not substitute for the originals.
- **Project-specific application** lives elsewhere: `oss/docs/product-framing.md`, `docs/private/product-hypotheses.md`, `docs/private/product-hypotheses/H-xxx.md`.

## Top 10 books for a lifetime

A compressed shortlist. If you only read ten books on PM, read these.

1. **Marty Cagan** — *Inspired*
2. **Richard Rumelt** — *Good Strategy / Bad Strategy*
3. **Teresa Torres** — *Continuous Discovery Habits*
4. **Clayton Christensen** — *The Innovator's Dilemma* (+ the JTBD essays)
5. **Eric Ries** — *The Lean Startup*
6. **April Dunford** — *Obviously Awesome*
7. **Donella Meadows** — *Thinking in Systems*
8. **Madhavan Ramanujam** — *Monetizing Innovation*
9. **Ryan Singer** — *Shape Up* (free online, basecamp.com/shapeup)
10. **Ben Horowitz** — *The Hard Thing About Hard Things*

**Aperitif:** Ben Horowitz — "Good Product Manager / Bad Product Manager" (1997 Netscape memo). Two pages. The single most-cited PM document. Calibrates taste in five minutes.
