# Validation and Shipping

Turning hypotheses into shipped, learned-from artifacts.

## Canonical sources

### Lean / validation

- **Eric Ries** — *The Lean Startup*. Build-measure-learn loop; MVP; validated learning; innovation accounting.
- **Steve Blank** — *The Four Steps to the Epiphany*, *The Startup Owner's Manual*. Customer development (the "discovery" precursor to Lean Startup).
- **Ash Maurya** — *Running Lean*, *Scaling Lean*. Lean Canvas (a one-page business model).
- **Dan Olsen** — *The Lean Product Playbook*. Six-step Product-Market Fit pyramid: target customer → underserved needs → value proposition → feature set → UX → PMF.
- **Melissa Perri** — *Escaping the Build Trap*. The distinction between output (shipped features) and outcome (moved metrics).

### Shipping rhythm

- **Ryan Singer** — *Shape Up* (basecamp.com/shapeup, free). Fixed time / variable scope. Appetite instead of estimate. Shaped work. Betting table. 6-week cycles + 2-week cooldowns. Antithesis to Scrum.
- **Colin Bryar & Bill Carr** — *Working Backwards*. Amazon's method from insiders. PR-FAQ, 6-pager, single-threaded leaders, bar-raiser.
- **Shreyas Doshi** — LNO framework (Leverage / Neutral / Overhead); "high-agency" concept; product sense essays.

## Core frameworks

- **Build-measure-learn.** The loop, not the linear plan. Speed is measured by time-through-loop, not throughput per phase.
- **MVP.** Minimum viable product: the smallest artifact that tests the riskiest assumption. Not "first version with bad UX."
- **Lean Canvas.** One-page: problem, customer segments, UVP, solution, channels, revenue, cost, key metrics, unfair advantage.
- **PMF pyramid (Olsen).** Bottom-up: target customer → underserved needs → value prop → feature set → UX.
- **Shape Up vocabulary.** *Appetite* = how much time you're willing to spend before you decide whether to continue. *Shaped work* = a problem chewed to a specific level of detail. *Betting table* = decision forum for the cycle.
- **PR-FAQ.** Write the press release and customer FAQ *before* building. If you can't write a compelling announcement, the product isn't ready to be built.
- **6-pager.** Narrative memo read silently at the start of a meeting. Forces the author to actually think.
- **Innovation accounting (Ries).** Metrics for early-stage work: different from normal accounting; track learning velocity.

## Practical principles

- **Define kill criteria before experiments start.** "What would we have to see to stop?" If you can't answer, you'll never stop.
- **Appetite > estimate.** Estimates are probabilistic forecasts of unknown scope. Appetite is a time budget against which you cut scope.
- **Write the output first.** Before building the report, write the report. Before building the feature, write the announcement. Exposes whether the thing is actually coherent.
- **Small experiments, large interpretation.** It is easier to ship 10 cheap experiments than 1 expensive one; the bottleneck is usually interpretation and decision-making, not experiment throughput.
- **Cooldowns are load-bearing.** The team's ability to fix, research, refactor, and rest during cooldowns is what keeps shaping cycles high-quality.
